# AiWB 财务 War Room 产品说明书（PRD）

版本：`v0.1`  
日期：`2026-04-17`  
状态：`需求固化中（可开发）`

## 1. 项目目标
在当前阶段，AiWB 的目标是基于 Google Sheets 完成财务逻辑共识与公式校验，形成可审计、可解释、可批量同步的协作系统。

核心目标：
1. 先把全部 Sheet 的业务逻辑和公式关系跑通并被团队确认。
2. 以 Python 作为公式权威源，反写公式/结果到 Google Sheets，避免双边维护。
3. 在逻辑确认前，以本地草稿为主，减少误同步风险。
4. 为未来迁移 AWS（多项目管理）保留架构升级空间。

## 2. 当前阶段范围（Phase 1）
### 2.1 In Scope
1. 全工作簿数据同步到本地 DataFrame（一次拉取，多次本地编辑）。
2. 全 Sheet 统一主键 `AiWB_UID`（缺失时本地补齐并标记待同步）。
3. 本地影子逻辑计算、对比、审计、增量回写。
4. Streamlit 可视化编辑与同步确认流程。
5. Apps Script 逻辑向 Python 迁移并统一执行入口。

### 2.2 Out of Scope（当前不做）
1. AWS 云端部署与多项目租户管理。
2. 同事并发协作权限精细化（待逻辑定型后再开放）。

## 3. 核心业务原则（已确认）
1. 主键统一：所有 Sheet 必须落地 `AiWB_UID`。
2. 公式权威：以 Python 逻辑为准，同步时反写到 Google Sheets。
3. 一致性标准：必须 `100%一致`，无容差。
4. 批次识别：按 `Final Detail` Sheet 的 `Final date` 计算批次。
5. 最终决算差异来源：`Draw Invoice List` Sheet。
6. 数值精度：
   - 百分比保留小数点后 `8` 位。
   - 金额保留小数点后 `2` 位。

## 4. 系统架构（三层模型）
1. 适配层（Data Adapter）
   - 封装 Google Sheets API 调用。
   - 负责拉取 values、拉取公式、批量更新与批量写回。
2. 逻辑层（Shadow Logic）
   - 在 Python 中复刻财务公式与 Apps Script 逻辑。
   - 输出计算结果、差异项、待同步单元格集合。
3. 校验层（Consistency Checker）
   - 对比 Python 计算值与 Sheets 当前值。
   - 输出红色高亮冲突行、差异摘要、同步前确认清单。

## 5. 数据与同步策略
### 5.1 数据加载
1. 使用 `@st.cache_data(ttl=600)` 缓存拉取结果。
2. 采用“本地快照 + 手动强刷”机制，避免每次刷新都全量重新拉取。
3. 拉取内容分离：
   - 编辑区显示值（display values）。
   - 公式独立缓存（用于公式检查与展示）。

### 5.2 本地草稿永久化
1. 编辑数据先写入本地草稿（磁盘持久化）。
2. 页面刷新或重启后可恢复草稿，不丢修改进度。
3. 同步到云端前必须经过差异确认与审计记录。

### 5.3 增量回写
1. 禁止逐单元循环调用 update。
2. 仅提交 Delta（变更单元格）并用 batchUpdate/values.batchUpdate 写回。
3. 写回前执行安全校验（目标工作簿标识/首行签名）。

## 6. 关键计算框架（基于投入法）
1. 核心方法：Cost-to-Cost Input Method（HKFRS 15）。
2. 主逻辑：
   - 累计收入 = （累计可计量成本 / 最新 EAC）× 最新 TP
   - 当期收入 = 本期累计收入 - 上期累计收入
3. 已确定约束：
   - 措施费、管理费不进入核心 POC 分母（措施费零毛利单独处理）。
   - Overrun：TP 不变，EAC 上调。
   - Savings（批次结算）：TP 与 EAC 同步下调，并用累计追溯法平滑。
   - 尾期 100% 完工倒挤，最终决算差异在末期调整。

## 7. UI/交互要求（当前基线）
1. 右侧为主编辑区（白色区域最大化）。
2. 工作表选择使用按钮组，位于右侧编辑区上方。
3. 左侧状态栏支持折叠。
4. 页面缩放目标为当前视觉密度约 `80%`。
5. 表格显示值；选中单元格时显示对应公式（Sheet 名后方或编辑区顶部）。
6. 状态指标（版本号、配额、同步时间等）放在较下方区域。

## 8. 审计与可追溯
1. 同步前展示变更摘要（新增/修改/删除行、金额差异汇总、实体维度汇总）。
2. 同步成功后写入本地 `aiwb_audit.log`：
   - 时间戳
   - 操作人（邮箱 + 英文名）
   - 变更统计
   - 影响 Sheet 与关键字段

## 9. 升级预留（Phase 2）
为 AWS 迁移预留以下抽象边界：
1. 适配层接口化：Google Sheets Adapter 可替换为 RDS/Redshift Adapter。
2. 逻辑层纯 Python 化：与 UI、存储解耦。
3. 审计层标准化：日志结构可直接接入云端日志系统。
4. 项目维度隔离：预留 `project_id` 上下文，支持多项目并行。

## 10. 待确认清单（阻塞项）
以下项已明确“后续补充”，当前先占位：
1. “措施费零毛利”具体落地 Sheet 与列名。
2. Overrun 与 Savings 的驱动字段（人工标记列名/规则）。
3. Draw Invoice List 中“最终决算金额”具体列名与数据格式。
4. Final Detail 中 `Final date` 的批次切分规则细则（按月/按批次窗口/按完工组）。

## 11. 验收标准（Phase 1）
1. 所有纳入范围的 Sheet 均有 `AiWB_UID`，且可双向追踪。
2. Python 影子逻辑与 Google Sheets 输出结果 `100%一致`。
3. 本地草稿可持久恢复，不因刷新丢失。
4. 同步仅写回变更单元格，不覆盖同事潜在输入列。
5. 审计日志完整可回放。
