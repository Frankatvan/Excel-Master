# AiWB 产品说明书 (Product Specification & Roadmap) v1.0

## 1. 产品定义
AiWB (AI Working Board) 是一个专为建筑行业 GMP 合同设计的**自动化审计与财务核算中台**。它旨在将混乱的原始施工成本数据（WBS/Payable/Final Detail），通过严谨的会计规则（HKFRS 15），转化为可被四大审计师（德勤）直接认可的财务报表与对账底稿。

---

## 2. 核心功能模块

### M1: 数据适配引擎 (Data Ingestion)
- **输入源**：接入 WBS 系统的 Payable (支付)、Unit Budget (预算)、Final Detail (结算) 原始数据。
- **GMP 过滤器**：基于 `Scoping` 表 E 列标记，实现总包与非总包成本的物理隔离。
- **主键同步**：全表强关联 `AiWB_UID`，确保每一分钱从原始账单到 109 汇总行的双向穿透。

### M2: 重分类审计大脑 (Classification Engine)
- **Rule ID 系统**：固化德勤认可的 ROE/ACC/RACC 核算规则。
- **自动打标**：一键对数万行原始记录进行“审计重分类”。
- **R251 还原逻辑**：根据“实际结算日期”自动触发历史成本还原，消除毛利波动。

### M3: 109 语义核算中枢 (Semantic Calculation)
- **语义映射**：基于 `Scoping` 表定义的 1~6 类科目，动态驱动 109 表的公式生成。
- **白盒公式链**：Python 不直接填数，而是生成“人可读”的 Excel 公式，保留审计复核痕迹。
- **5步法核算**：严格执行 Overrun 与 Savings 的动态对冲逻辑，自动计算 POC 与确认收入。

### M4: 全域监控与对账看板 (Meta Dashboard)
- **Meta DB**：在本地/云端建立元数据库，存储 100+ 项目的核心财务指纹。
- **动态查询**：在 Streamlit 界面实现多项目进度对比、风险 Rule ID 预警。
- **对账调节**：AI 辅助完成甲方付款与投入法收入的“差异调节表”。

---

## 3. 实施大节点 (Implementation Milestones)

### 第一阶段：单项目逻辑闭环 (Single-Project Blueprint)
*   **目标**：在 109 表模板中，实现“一键重分类 -> 一键更新公式 -> 结果 100% 正确”。
*   **交付物**：固化的 Rules ID 库、109 语义映射原型、审计可回溯的 Excel 模板。
*   **状态**：**当前阶段 (进行中)**。

### 第二阶段：自动化与安全加固 (Automation & Security)
*   **目标**：实现“傻瓜式”操作，防止误改，并具备“吃后悔药”的能力。
*   **交付物**：Google Sheets 区域锁定功能、版本快照/一键回滚系统、异常数据前端高亮报警。

### 第三阶段：多项目聚合与 Meta DB (Scaling)
*   **目标**：从“一个 Sheet”变成“100 个 Sheet 的指挥部”。
*   **交付物**：本地 SQLite 数据库、多项目看板（Streamlit）、跨项目财务指标搜索引擎。

### 第四阶段：云端迁移与 API 闭环 (Cloud Ready)
*   **目标**：彻底摆脱手动导入。
*   **交付物**：WBS API 自动对接、AWS 云端多租户部署、月度连续跟踪版本。

---

## 4. 审计合规性承诺 (Compliance Assurance)
- **HKFRS 15 兼容性**：所有 POC 计算与收入确认逻辑均经过德勤备忘录校验。
- **可审计性**：生成的 109 表每一格都有据可查，提供“分录调整建议表”作为最终指导。
- **数据一致性**：Python 本地核算结果与 Excel 显示结果必须 `0 容差`。

---

## 5. 待办与阻塞项 (Blockers)
1. **对账算法原型**：需要针对“乱序甲方对账”开发专门的 AI 匹配策略。
2. **德勤最终确认**：Rules 规则库的最终冻结。
3. **WBS API 文档**：为 Phase 4 预留的接口规范。
